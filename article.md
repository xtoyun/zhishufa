# 当 AI 学会"看"汉字的重心：用计算机视觉破解书法之道

> 千年以来，书法家靠直觉判断一个字是否"稳"。现在，我们用像素级的计算，把这种直觉变成了可视化的数据。

---

## 一、一个书法老师永远不会告诉你的秘密

拿起毛笔写"永"字，你觉得写得还行。但老师看了一眼，说："歪了。"

你盯着看了半天——明明每一笔都照着字帖来的，到底歪在哪？

答案藏在四个字里：**视觉重心**。

物理重心是笔画质量的加权平均，视觉重心却是人眼"觉得"的中心。两者经常不一致——一个左撇舒展的"我"字，物理质心偏左，但人眼因为它右侧的斜钩和点画，会"觉得"重心在右边。

这种不一致正是书法的魅力所在，也是初学者最痛苦的地方：**你看不见自己的问题，因为问题不在笔画本身，而在笔画之间的关系。**

这就是 AI 可以帮上忙的地方。

---

## 二、启功的"黄金律"和阿恩海姆的"张力"

在进入代码之前，我们先搞清楚一件事：视觉重心到底由什么决定？

**启功先生**在《论书绝句》中提出"结字黄金律"：一个字好不好看，关键看中心部位——也就是书法行话里的"中宫"——笔画是否紧凑。中宫紧则字有精神，中宫散则字无骨力。用现代语言说：**笔画密集的区域，其视觉权重大于稀疏区域。**

**鲁道夫·阿恩海姆**（Rudolf Arnheim）在《艺术与视知觉》中提出了另一个维度：倾斜会产生方向性"张力"。横画略微向右上方倾斜，人眼就会感受到一种向上的力；撇画向左下伸展，则产生下拉的力。这些力会"拉扯"视觉重心。

**姜夔**在《续书谱》中则讲"真态"——每个字有其天然的笔画分布，顺其自然即是美，强行对称反而呆板。

把这三家理论放在一起，我们就得到了视觉重心的三个决定要素：

| 要素 | 通俗含义 | 理论来源 |
|------|---------|---------|
| **分量** | 笔画越多越"重"的地方，重心往那里靠 | 基础物理学 |
| **中宫** | 笔画越密集的地方，视觉吸引力越强 | 启功"黄金律" |
| **斜度** | 笔画的方向会产生"拉力"，牵引重心偏移 | 阿恩海姆"视知觉张力" |

再加上一个人类视觉系统的固有偏差：**人眼的视觉中心系统性偏上约 5-8%**（这是格式塔心理学反复验证过的结论——同样大小的圆，放在几何中心会"看起来偏低"）。

四个维度凑齐，就可以开始建模了。

## 三、用 Canvas 把汉字变成数字

第一步，我们需要让计算机"看见"一个汉字。

做法很直接：用 HTML5 Canvas 把文字渲染成位图，然后逐像素扫描。关键细节在于：

```text
不是简单地判断"有墨/无墨"
而是：
  darkness = alpha / 255          ← 墨色深浅
  weight = darkness × (1 - R/255) ← 消除 RGB 子像素干扰
```

这样，抗锯齿边缘的半透明像素不会突然丢失，它们的权重自然降低，整个计算就更平滑。

有了每个墨色像素的坐标和权重 `(x, y, weight)`，物理质心就是标准的加权平均：

```
rawCX = Σ(weight × x) / Σ(weight)
rawCY = Σ(weight × y) / Σ(weight)
```

这是起跑线，不是终点。

## 四、中宫热力——让"密集"变成数字

中宫收紧是书法结构里最难量化的概念。我们用了这样一个方法：

将 480×480 的画布划分为 **28×28 的网格**，每个格子累加落在这个区域内的墨色像素权重。然后：

```
localDensity = cellWeight / maxCellWeight   ← 归一化到 0~1
enhancedWeight = weight × (1 + localDensity × 2.0)  ← 密集区放大
```

用这个增强后的权重重新计算一次质心，就得到了"密度感知质心"。

效果是什么？以"不"字为例——它的竖画和右点都集中在右侧偏下的位置，中宫热力会在那里形成一个高亮区。密度质心相较物理质心会明显向那个方向偏移。

这正是书法家眼睛在做的事：**大脑自发给密集区域分配更多注意力权重。** 我们用 `densityGrid` 和放大系数模拟了这一步。

右侧面板那个 280×280 的**中宫热力图**就是这组数据的可视化——从浅黄（稀疏）到深红（密集），一眼就能看出汉字的"灵魂区域"在哪里。

## 五、斜度感知——笔画的方向会"拉"重心

这是整个模型里最微妙的部分。

一个横画多的字（如"三"），所有横画都向右上倾斜。阿恩海姆说这种倾斜产生向上的张力。计算机怎么看？

我们用**类 Sobel 梯度检测**——对墨色像素抽样，计算每个采样点周围像素的亮度梯度：

```
Gx = 左右像素亮度差    → 检测垂直边缘
Gy = 上下像素亮度差    → 检测水平边缘
```

- 如果 `|Gx| 远大于 |Gy|`：这是一个水平边缘（横画的上下边界），它暗示笔画沿水平方向延伸，贡献于纵向重心偏移
- 如果 `|Gy| 远大于 |Gx|`：这是一个垂直边缘（竖画的左右边界），贡献于横向重心偏移
- 其他：斜向笔画，双向贡献

举例：一个横画很多的字，水平边缘大量聚集，`avgSlantY` 会是一个负值（假设笔画在画布上半部分）。这个值会轻轻地把重心往下"拉"——抵消横画给人的向上飞升感。

## 六、融合——三个要素变成"一个红点"

三个要素算出三个质心位置后，通过加权融合变成一个最终的视觉重心坐标：

```
visualCX = blended + densityShift + slantShift + rightBias(2.5% 字宽)
visualCY = blended + densityShift + slantShift - upBias(6.0% 字高)
```

五个参数全部可通过滑块实时调节：

| 参数 | 默认 | 含义 |
|------|------|------|
| 笔画分量权重 | 70% | 物理质心→密度质心 的混合比例 |
| 中宫收紧权重 | 60% | 笔画密集区对重心的吸引力强度 |
| 斜度影响权重 | 40% | 笔画方向的影响力 |
| 视觉偏上偏差 | 6.0% | 补偿人眼"视觉中心偏上" |
| 视觉偏右偏差 | 2.5% | 补偿书写节奏的"先小后大"偏好 |

当你拖动"中宫收紧权重"从 0 拉到 100，屏幕上那个朱红色的圆点会真实地向笔画密集区滑过去。这就是抽象书法理论的物理可视化。

## 七、切换字体——同一个字，不同的"心"

这是最让书法学习者有启发的功能。

同一个"永"字，切换到**楷书**，重心可能是 (242, 238)；切换到**宋体**，可能变成 (237, 244)；切换到**黑体**，又变了。

为什么？因为不同字体的笔画设计理念不同：

- **楷书**：模拟手写，笔画起收分明，骨肉匀停，重心往往接近中宫
- **宋体**：横细竖粗，竖画在视觉上"更重"，会把重心往竖画集中的区域拉
- **黑体**：笔画均匀，没有粗细变化，重心最接近几何中心
- **仿宋**：楷体骨架 + 宋体笔形，是两者的折中

这揭示了一个深刻的道理：**"好看"不是绝对的。** 同样的结构原则，在不同字体里需要微调。书法学习也是如此——你学颜体需要关注的重心偏移量，和学欧体完全不同。

系统支持 7 种字体（霞鹜文楷、鸿雷拙书、鸿雷板书三个 Web 字体 + 楷/宋/黑/仿/明/思源宋六个系统字体），实时切换、即时重算。

## 八、书法学习者的实操指南

说了这么多理论，这个东西到底怎么用？

### 场景一：自查"字歪了没"

写一个字，拍照，输入到工具里（或者直接选预设字）。看三个指标：

1. **偏移量 (Δx, Δy)**：如果绝对值小于 5px，恭喜，你的结构感不错
2. **上:下 比例**：如果上半超 55%，可能该收一收了
3. **解读文案**：AI 自动结合偏移量和书法理论给出的点评

### 场景二：理解"中宫"

初学者最常犯的错就是笔画散。切换到**密度热力模式**，看中宫热力图——如果中心区域是浅黄色甚至空白，说明你的笔画全散在外围了。对比字帖中同一个字的热力图，差异一目了然。

### 场景三：感受"斜度"

输入"飞"字，把斜度影响权重调到 0，再调到 100。你会发现重心位置有明显的垂直位移。这是因为"飞"字的横折斜钩方向复杂，斜度修正揭示了"方向感"对重心的真实影响。

### 场景四：对比自己的字和名家的字

这是最实战的用法——用截图工具截取你写的字和字帖上的字，分别以相同的字体渲染分析（系统字体可以近似），对比两者的重心坐标、上下比例和热力分布。**从"感觉不对"到"数据告诉你偏差在哪里"**——这就是学习效率的根本提升。

## 九、技术栈与开源

整个工具是一个**单文件 HTML**应用，约 1400 行代码，零后端，零框架依赖（仅 CDN 引入 Vue 3 做响应式）：

- **Canvas 2D API**：文字渲染 + 像素分析 + 图形标注
- **Web Font Loading API**：确保字体就绪后再做 Canvas 分析
- **离屏渲染**：分析在隐藏 Canvas 上进行，不干扰用户看到的画面
- **150ms 防抖 + nextTick 异步**：保证交互流畅，输入不卡顿

所有计算在浏览器本地完成，不上传任何数据。
